Various informations:

The left end neuron layer makes the input neuron layer.
The right end neuron layer makes the output neuron layer.

Every time you unvalidate and revalidate a neural network, you lose all its apprenticeship. Processing mode on a non-learned neural network works but make no sense. (Thanks captain obvious!)

The multithreading functionnality can significantly increase the learning speed only if used on big amount of data and/or big neural networs. Else, used on small amount of data and small neural networs, the multithreading can slow dows the learning speed.

The learning algorithme stops when the MSE does not changes enough during _insufficientProgressionsNeededToStop iterations. The last interration progression (LIP) must be _insufficientProgressionsNeededToStop times in a row under the threshold given in the "Minimum progression per iterations" field. Setting the value of 0 for the threshold is equivalent to disable this funtionnality.

The learning mode "Learn with a control sample" will make the neural network only learning the first half of the data in order to comute a potential error on new non-learned data. The comparing between the results obtained on the learned data and the non learnd data can be read in the displayed end learning message.

Learning mode:

To enter in the learning mode :
  • have a validated perceptron
  • load a table (a CSV file)
  • you must have : Ncolumns in the table = Ninput neurons + Noutput neurons

Processing mode:

To enter in the processing mode :
  • have a validated perceptron
  • load a table (a CSV file)
  • you must have : Ncolumns in the table = Ninput neurons
After using the processing mode, the "Export this Table (as CSV File)" option int the "File" menu will be unlocked until you do something.
